# Phase 1: SIGSYS-driven preload (design)

This branch (`phase1/sigsys-preload`) is the staging ground for the
v0.2 wrapper redesign that replaces ptrace-based syscall interception
with a `SECCOMP_RET_TRAP` + SIGSYS handler running entirely in the
wrapped process.

## Why

The ptrace transports are correct but slow on chromium-class
workloads — every intercepted syscall costs 5-10 kernel context
switches. SIGSYS keeps the kernel-side cost (per-syscall signal
delivery) but eliminates the userspace round-trip to a separate
tracer process. For libc-routed syscalls the libc shim short-circuits
even the SIGSYS detour.

## Three-layer split

```
preload/
├── core/                  layer 1: freestanding C, async-signal-safe
│                          knows about: tunnel fds, fdproxy.sock,
│                          shared state, dispatch table, SIGSYS handler,
│                          seccomp filter
│   ├── syscall.h          inline-asm syscall wrappers, no libc
│   ├── syscall.S
│   ├── shared_state.c     mmap'd uwgshared.Table consumer
│   ├── fdproxy_sock.c     raw protocol to /tmp/uwgfdproxy.sock
│   ├── seccomp.c          BPF filter installation
│   ├── sigsys.c           handler entry, ucontext decode → dispatch
│   ├── dispatch.c         table: nr → uwg_*
│   ├── socket_ops.c       socket(), socketpair(), close()
│   ├── bind_ops.c         bind(), listen(), accept(), accept4()
│   ├── connect_ops.c      connect()
│   ├── msg_ops.c          send/recv*msg / send/recvfrom / send/recvmmsg
│   ├── stream_ops.c       read/write/readv/writev with TCP-stream fast path
│   ├── fd_ops.c           dup/dup2/dup3, fcntl, getsockname etc.
│   └── sendfile_ops.c     sendfile/splice/vmsplice/tee/copy_file_range
│
├── shim_libc/             layer 2: thin libc wrappers (only in .so build)
│   ├── shim.c             one wrapper per overridden libc(2) function
│   │                      → calls into core dispatch
│   │                      → wraps -errno-as-i64 → -1+errno boundary
│   └── shim_init.c        __attribute__((constructor)) → core init
│
└── shim_dns/              layer 3: glibc DNS interception (only in .so)
    └── dns.c              getaddrinfo, gethostbyname etc.
                           → core CONNECT/resolve flow
```

Both shims are dumb wire-format adapters. The core knows tunnel state
and is the ONLY place that consults the shared table. Both shims call
the SAME `uwg_recvmsg`/`uwg_connect`/etc. functions; only how they
arrived (libc-hook vs SIGSYS-decode) differs.

## Async-signal-safety constraint (load-bearing)

The SIGSYS handler can re-enter while another thread is already
mid-handler. Therefore the core layer must:

- Use ONLY inline-asm syscall instruction (no libc syscall wrappers
  that touch TLS errno).
- Use ONLY `mmap`-allocated memory (no malloc; preallocate at init).
- Use ONLY atomics + futex for sync (no pthread mutex — they touch
  malloc internals via PTHREAD_MUTEX_INITIALIZER lazy init).
- Use ONLY stack memory or pre-allocated per-thread arenas. Per-thread
  scratch buffer for fdproxy socket roundtrip lives in TLS, allocated
  at first-use from a pool.
- All atomic ops must be lock-free (no `_Atomic struct` larger than
  the platform's atomic-CAS width).

## Seccomp filter (trap list)

```
control-plane (always trap, dispatch via core):
  socket, socketpair, close, connect, bind, listen, accept, accept4,
  setsockopt, getsockopt, getsockname, getpeername,
  dup, dup2, dup3, fcntl, shutdown

message-style (always trap):
  recvfrom, recvmsg, recvmmsg, sendto, sendmsg, sendmmsg

stream-style (trap; core fast-paths TCP-stream, handles UDP-connected):
  read, write, readv, writev, pread64, pwrite64

stream-only-meaningful syscalls (trap; core returns -EINVAL for
  managed UDP datagram, passthrough for everything else):
  sendfile, splice, vmsplice, tee, copy_file_range
  [Phase 3 long-tail — kernel returns -EINVAL itself for datagram
  sockets, but our datagram fds are unix-stream sockets so we have
  to enforce the kernel's contract ourselves.]

execve (RET_TRACE — bootstrap supervisor handles it):
  execve, execveat

bypass-secret rule (always RET_ALLOW):
  any syscall with BYPASS_SECRET in arg6

dangerous-but-safe-to-noop (Phase 3):
  seccomp(2) (we already PR_SET_NO_NEW_PRIVS, but we can't be sure
              an app won't try; trap and lie),
  prctl(PR_SET_DUMPABLE, 0) (would prevent our ptrace bootstrap),
  prctl(PR_SET_NO_NEW_PRIVS, 0) (no-op; already set)

everything else: RET_ALLOW (kernel runs it directly)
```

## Bypass-secret protocol

Documented from existing uwgpreload.c (this is unchanged from current
behavior — the SIGSYS work just adopts the same primitive):

- A 64-bit secret is generated by the Go uwgwrapper at process startup
  and passed into the wrapped process via env var `UWGS_TRACE_SECRET`.
- The seccomp filter has a rule near the top: if `args[5] == SECRET`,
  return `SECCOMP_RET_ALLOW` immediately.
- Any syscall the core layer issues to "really do this in the kernel"
  goes through `passthrough_syscall(nr, a1, a2, a3, a4, a5, SECRET)`.
- Kernel sees the secret in arg6, filter allows, kernel runs the
  syscall normally.
- Args 6 is otherwise unused by Linux x86_64 syscalls (max 6 args,
  numbered 0-5 → only first 5 are user-controlled in practice; arg5
  is the 6th positional and is the choke point we use).

## Init sequence (matters — wrong order bricks the process)

```c
// preload/core/init.c::uwg_core_init() — called from
//   * shim_init.c constructor (the .so case)
//   * uwg_static_init() exported entry (the static-binary case, Phase 2)

1. read env (UWGS_TRACE_SECRET, UWGS_FDPROXY, UWGS_SHARED_STATE_PATH)
2. mmap shared_state
3. open fdproxy.sock (lazy — first use)
4. preallocate per-thread scratch arenas (TLS via __thread)
5. install sigaltstack (per-thread; for SIGSYS handler stack)
6. install SIGSYS handler:
     sigaction.sa_flags = SA_SIGINFO | SA_ONSTACK | SA_NODEFER
     sigaction.sa_mask  = SIGSYS  (block recursive SIGSYS)
     sigaction(SIGSYS, &sa, NULL)
7. prctl(PR_SET_NO_NEW_PRIVS, 1, 0, 0, 0)
8. seccomp(SECCOMP_SET_MODE_FILTER, SECCOMP_FILTER_FLAG_TSYNC, &prog)
   // TSYNC propagates to all current threads
   // Future threads inherit via clone() and the kernel's
   //   thread-clone seccomp inheritance.

// After return: every interesting syscall is now trapped.
// Kernel will SIGSYS us if we call one without the bypass-secret.
// But the libc shim catches all libc-routed syscalls before they
// reach the kernel, so SIGSYS only fires for inlined-asm leftovers.
```

## Existing uwgpreload.c migration plan

The existing file is 3368 lines. It is NOT being deleted in Phase 1;
it is being **decomposed** in place. Every helper / function in it
falls into one of the three layers above. The migration is mechanical:

1. Move the bypass-secret machinery to `core/syscall.c` +
   `core/syscall.h`. This is the foundation everything else depends on.
2. Move the shared-state table ops (`tracked_*`, `local_tracked_*`,
   `hot_path_*`) to `core/shared_state.c`.
3. Move the fdproxy IPC functions to `core/fdproxy_sock.c`.
4. Move the per-syscall handlers to `core/socket_ops.c`,
   `core/connect_ops.c`, etc. Each becomes a `uwg_*` returning -errno.
5. Replace the existing libc-hook bodies in `uwgpreload.c` with
   one-liners that call `uwg_*` and wrap -errno → errno+(-1). Move
   those one-liners to `shim_libc/shim.c`.
6. Move DNS hooks to `shim_dns/dns.c`.
7. After all moves, `preload/uwgpreload.c` becomes a thin
   "this file exists for backward compatibility" stub or is deleted
   entirely.

## What lands in this initial commit

This commit lays the foundation only — no migration of existing code.

- `preload/core/seccomp.c` — BPF filter generator (standalone, ~150 LOC)
- `preload/core/sigsys.c` — SIGSYS handler shell (decode ucontext,
  call dispatch stub which currently returns -ENOSYS)
- `preload/core/dispatch.c` — stub dispatch table
- `preload/core/dispatch.h` — function signatures (above)
- `preload/core/syscall.h` — inline-asm syscall wrappers
- `preload/PHASE1_DESIGN.md` — this doc

Tomorrow's commits will mechanically extract from existing
uwgpreload.c into core/ files. Build glue ensures both old and new
artifacts compile cleanly during migration.

## What's NOT in Phase 1

- Static-binary support (`uwgpreload-static` build) — Phase 2
- `sendfile` family hardening — Phase 3
- `seccomp(2)` and `prctl()` defensive interception — Phase 3
- execve bootstrap supervisor in Go uwgwrapper — Phase 2
- Removing ptrace mode entirely — long-tail; ptrace stays as fallback
  for environments where seccomp is disabled.
